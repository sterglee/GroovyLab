Right-associativity operators

Scala implements all operators as method calls. For example, if M is a Matrix then M + 10 is implemented as M.+(10). The opposite case 10 + M, is implemented by defining an implicit conversion that directs the compiler to transform the Int element 10, to a Matrix of compatible size with M and with all its elements set to 10, and then performing the matrix addition.
However, the implicit conversion implies an object transformation at run time and thus some computational cost. In order to avoid the overhead of the implicit conversion the right associativity of operators that end with ':' can be exploited. In particular, any method that ends in a ':' is invoked on its right operand, passing it to the left operand. For example, in
var m= rand(20, 30);    var r = 20 +: m
the addition is evaluated as: var r = m.+(20)
Right-associativity operators +:, -:, *: are supported by classes Vec, Matrix, Mat and for any Number (e.g. Int, Double) at the left side, i.e. the general pattern is:
   (Short | Int | Long | Float | Double) (+: | -: | *:) (Vec | Matrix | Mat)
