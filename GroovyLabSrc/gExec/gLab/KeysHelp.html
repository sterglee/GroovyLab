Help on the basic editor's keystrokes


F6:    executes selected text  or current line.

Shift-F6:    executes selected text  or current line with a separate thread .


F5:  clears the output console

F11:  replace abbreviations – The abbreviations list file is initialized at the first time F11 is used.

Double click on an identifier: Displays its value.                                

Ctrl-SPACE:     Code Completion, global completion when in “Global” mode, field completion using Java reflection when in “Groovy” mode

F2:   executes code up to cursor position 

F4:  displays a completion list for an identifier (Use ENTER or mouse double click  to select an entry ESC to clear it)  



Shift-F4:  displays a completion list for a class name, for example to selecting org.jblas.DoubleMatrix and pressing Shift-F4 inspects this class.

F12:  updates the GroovySci's variables binding. The updated variables are available for code completion.

F8 with selected text:  buffers the selected import statements

Ctrl-F8 with selected text:  buffers the selected code.


F9 with not any selected text:  compiles and execute a Java class consisted of the whole text in editor.

F9 with selected text that specifies some methods:  compiles and executes these methods as static methods of a Java class. Then these methods can be used from GroovyLab scripts. 

F7:  creates a stand alone Java application with the functionality of the Groovy script.

F10:  browses the properties of the class of a selected identifier


Select text (e.g. svd) and press F1 displays autocompletion info on that item.


Completion  Notes

The RSyntaxArea editor supports two modes of code completion, the Global mode and the Groovy mode.  The menu “Completion” can be used to switch between these two completion modes, with the submenu option: 
“ Toggle Global/Groovy completion mode, current mode is: .... “
That submenu also displays the current completion mode. Also the current completion mode is saved upon GroovyLab exit and is available for the next working session.

The “Global” completion mode is useful to remind the large number of  routines of GroovyLab. Also,  many of these routines are overloaded and thus it is more difficult for the user to remember their usage. One good example is the plot() routines that has a lot of overloaded versions. The user can simply type plot and then CTRL-SPACE, while RsyntaxArea is in “Global” completion mode. The code completion feature of the RsyntaxArea editor reminds the user of all the overloaded versions of plot.
Also, we can from the “Completion” menu expand the completion lists available at the “Global” completion mode using information extracted from libraries using Java reflection. 

The “Groovy” completion mode can also be very useful for other completion tasks.  That mode extracts information using Java Reflection. 
Very convenient is the member completion feature. Suppose that we execute the command:
 xx = new JFrame("test")
Then we can easily remember the many fields/methods of the javax.swing.JFrame class, using:
xx. <CTRL-SPACE>
